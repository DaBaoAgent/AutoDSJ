# DY 工作流 · 视觉索引识别精度重构 开发报告

> 目标：让视觉索引"看清"每一帧 —— **谁**（哪个演员/角色）、**在干嘛**（说话/动作）、
> **在哪**（场景）、**有什么细节**、**旁边是谁在做什么**。当前"张冠李戴 / 描述发糊"
> 的根因在识别**上游**，本报告给出根因诊断 + 三条重构方案 + 分期落地计划。
> 编写日期：2026-07-12　作者：Hermes Agent（徐黑蛋号）

---

## 一、问题诊断：为什么现在"看不清"

逐帧读了识别链路（`backend/vision_api.py` / `backend/drama_source_index.py` /
`backend/visual_matcher.py` / `backend/embed_match.py`），5 个根因，全部在**识别上游**，
下游的语义向量匹配再强也救不回来（"garbage in, garbage out"）：

| # | 根因 | 位置（file:line） | 后果 |
|---|------|------------------|------|
| 1 | **抽帧分辨率只有 480×270** | `vision_api.py:162‑165` `_extract_frames` 里 `scale=480:270` | 一张 480×270 里人脸只有 30~60px，VL 模型根本看不清五官/表情/道具细节，更别提认演员。 |
| 2 | **一次塞 8 帧给模型** | `dy.py:110` `visual_batch_size=8` → `drama_source_index.py:132` | VL 的注意力被 8 张图稀释，每帧只能给一两句笼统话，"旁边是谁在做什么"这种细节被抹平。 |
| 3 | **固定间隔抽帧 4~12s（目标 320 帧）** | `drama_source_index.py:475‑488` `adaptive_frame_interval` | 45 分钟一集 ≈ 8s/帧。一个镜头常只抽到 1 帧甚至跨两个镜头，漏掉关键动作与换人瞬间。 |
| 4 | **完全没有演员参考图** | `vision_api.py:258‑283` `_call_bailian_vision` 只发 `prompt + 帧图` | prompt 里写"能确认姓名就写姓名"（`_vision_prompt` `vision_api.py:196‑205`），但模型**没有"刘亦菲长什么样"的锚**，永远无法确认 → 全部退化成"穿红裙的年轻女子"。**身份链从源头就是空的。** |
| 5 | **身份机制依赖模型自己吐"演员（饰演角色）"** | `visual_matcher.py:96‑135` `_ACTOR_ROLE_RE` / `register_character_aliases` / `_NICKNAME_BRIDGE` | 上游（根因 4）从没产出这种标注，所以 `_DYNAMIC_CHAR_ALIASES` 基本为空，全靠"默认主角=玫瑰"和昵称桥接硬兜 → 这正是**张冠李戴**的直接来源。 |

补充：`caption` 字段是下游 **embedding 语义匹配的唯一语义源**
（`embed_match.py` + `anchored_pipeline.allocate_visual_all`）。caption 越糊，
"解说→画面"越飘。**精度是上游决定的**。

## 二、重构总思路

**把"身份"和"内容"拆开，各用最擅长的工具，再融合成一条高质量结构化帧记录。**

- **身份（谁）** → 人脸识别做，别让 VL 猜。这是准度最大的杠杆。
- **内容（在干嘛/在哪/细节/旁边谁）** → VL 做，但要给它高清图 + 每次少帧 + 把已知人名喂进去。
- **融合** → 每帧产出结构化对象：`people:[{role, box, prominence, is_speaking}]` +
  `caption / scene / action / props / emotion / shot_scale`，caption 里显式写
  "黄亦玫（居中，正在说话）对庄国栋…"，下游向量匹配立刻变准。

## 三、三条方案

### 方案 A（强烈推荐 · 主力）：本地人脸库 + 高清抽帧 + 强 VL 分工

这正是你说的"主演人脸照片当参考图比对"，但用**最快最准**的方式落地 ——
不是每帧都把参考图喂给 VL（慢、贵、还不准），而是用本地 **InsightFace(buffalo_l / ArcFace)**
离线比对人脸向量。

- **A1 建演员人脸库（一次性，全剧复用）**
  每部剧建目录 `_faces/<角色名>/*.jpg`，每个主演放 3~5 张清晰正脸（不同角度/表情）。
  跑 `dy faces build` 用 ArcFace 抽 512 维人脸向量存 `_face_gallery.json`。
  《玫瑰的故事》8 位主演建一次，38 集全复用。
- **A2 每帧人脸识别（本地、快、准、零 API 费）**
  对每张抽帧 detect + embed，与 gallery 算余弦（阈值 ~0.35~0.40）→ 得到：
  画面里有**谁**（角色名 + 置信度）、**谁是主体谁在旁边**（人脸框大小/位置 → `prominence`）、
  **谁在说话**（居中 + 大脸 + 景别近 → `is_speaking` 启发式）。
- **A3 高清抽帧**：`480×270` → **1280×720**（人脸 150~300px，ArcFace 才可靠，VL 也看得清细节）。
- **A4 VL 只负责"内容"，且把已识别人名注入 prompt**
  prompt 改为："画面中已知人物：黄亦玫(居中)、庄国栋(右侧)。请描述动作/场景/道具/情绪/景别/谁在说话，
  不要臆测未列出的身份。" VL 不再瞎猜身份，**每次只发 1~2 帧**，描述深度大增。
- **A5 融合**成结构化帧记录，写回 `_source_visual_index.json`；`caption` 由身份 + 内容拼成人话。

**收益**：身份准度从"基本靠猜"跃到 ArcFace 级（明星正脸场景 >95%）；内容准度靠高清 + 少帧 + 已知人名大幅提升；
`visual_matcher` 的 `register_character_aliases` 终于有**真数据**可吃，`_NICKNAME_BRIDGE` 继续桥接昵称。

### 方案 B（可叠加）：镜头边界抽帧替代固定间隔

用 ffmpeg `scdet` / `select='gt(scene,0.3)'`（本机 ffmpeg 已支持，已验证）检测镜头切换，
**每个镜头取代表帧**（`thumbnail` 滤镜选最具代表性的一帧），而非固定 8s。
保证"每个不同镜头至少 1 帧、不跨镜头糊成一团"，还能在总帧数不变的前提下把帧"花在刀刃上"，
抵消 A3/A4（高清 + batch=1）带来的调用量上升。可与 A 组合。

### 方案 C（无本地依赖时的退路）：纯 VL 参考图路线

不想装 InsightFace 时：直接用 Qwen‑VL 多图能力，每次请求 = N 张主演参考图（带角色名标签）+ 1 张抽帧，
prompt 让模型"只在参考图中的人出现时报角色名"。比现状强很多，但**每帧都要重传参考图 token
→ 更慢更贵**，且身份精度不如 ArcFace。作为 A 的 fallback。

## 四、方案对比（推荐 A + B）

| 维度 | 现状 | A 本地人脸库 | B 镜头边界 | C 纯VL参考图 |
|------|------|-------------|-----------|-------------|
| 身份精度（谁） | ★☆☆☆☆ 靠猜 | ★★★★★ ArcFace | —（配合A） | ★★★☆☆ |
| 内容精度（干嘛/细节） | ★★☆☆☆ | ★★★★☆ 高清+少帧 | ★★★★☆ 不漏镜头 | ★★★☆☆ |
| 速度 | 快（糊） | 中（本地人脸+VL少帧） | 快 | 慢（每帧传参考图） |
| API 成本 | 低 | 中（帧数↑但可用B压） | 低 | 高 |
| 新增依赖 | — | insightface+onnxruntime(CPU) | 无(ffmpeg已具) | 无 |
| 实施量 | — | 中 | 小 | 小 |

本机无 NVIDIA GPU（`nvidia-smi` 不存在），InsightFace 走 **onnxruntime CPU**：
buffalo_l 检测+识别约 50~150ms/帧，一集 300~600 帧 ≈ 1~2 分钟，可接受。

## 五、落地改动点（file:line 级）

1. **新文件 `backend/face_gallery.py`**：`build_gallery(faces_dir)` / `identify(frame_path)` /
   余弦比对；模型 buffalo_l，首次自动下载缓存。
2. **`backend/schemas.py:14`** 新增 `VisualSettings`：
   `frame_width=1280, frame_height=720, batch=1, use_face_gallery=True,`
   `faces_dir="_faces", face_threshold=0.38, scene_detect=True`（挂到 `AppSettings`）。
3. **`vision_api.py:162‑165`** `_extract_frames`：分辨率参数化（读 `VisualSettings`），
   480×270 → 1280×720；`-q:v 5` → `-q:v 2`（更清）。
4. **`vision_api.py:196‑205`** `_vision_prompt`：改为"注入已知人物 + 逐人结构化"，
   新增 `people[]`（含 `is_speaking`/`prominence`）、`props` 字段。
5. **`vision_api.py:258‑283`** `_call_bailian_vision`：支持每帧 1 图 + 接收 identity 文本注入。
6. **`drama_source_index.py:125` `_annotate_source_frames`**：抽帧后**先跑人脸识别**，
   把 identity 写进每个 record，再调 VL；`batch_size` 默认 8 → 1（`dy.py:110` 同步）。
7. **`backend/visual_matcher.py:101` `register_character_aliases`**：直接吃 A2 的 `role` 名
   （现在有真数据），`_NICKNAME_BRIDGE` 保留。
8. **新 CLI（`dy.py`）**：`dy faces build [--folder]` / `dy faces add <角色> <图...>` /
   `dy faces list`；`dy status` 增加"人脸库 N 人/M 张"一行。
9. **匹配报告**：`★匹配报告.json` 的 `visual_match_evidence` 增加 `identified_people`，
   便于你逐句核对张冠李戴。

## 八、风险与验证

- **依赖安装**：`pip install insightface onnxruntime`（CPU）；首次跑自动下载 buffalo_l 模型（~300MB）。
  装进项目 `.venv`，`dy doctor` 增加自检项。
- **成本/耗时**：高清 + batch=1 会增加 VL 调用次数与耗时；用 P2 镜头边界抽帧压总帧数抵消。
- **阈值调参**：`face_threshold` 默认 0.38，正脸库质量差时会漏识别 → 支持 `dy faces` 补图重建。
- **侧脸/背影/远景**：ArcFace 对侧脸/背影会 miss，此时 identity 留空、交给 VL 描述外观 + 场景锚定兜底，
  不硬编造身份（比现在瞎猜更安全）。
- **验收标准**：第4集重跑后，`★匹配报告.json` 里"解说点名某角色→画面命中该角色"的比例显著上升，
  人工抽查身份正确率 >90%，且无"说 A 配 B 画面"的硬错。
