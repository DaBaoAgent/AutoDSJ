# 大镜头场景段锁定匹配（scene-segment locking）

比"逐帧语义匹配"更强的一层：把原片切成有意义的**大镜头场景段**（首次会议 687-889、
病房 484-575、活动现场 1620-1761…），解说按关键词/人物分类到场景，命中就**硬锁定**到
该场景的原片区间里匹配。解决"说到开会却放成食堂/活动现场"的张冠李戴。所有路径相对
`D:\@kaifa\AutoDSJ\project\`。

## 何时用
- 用户反馈某类解说画面成片系统性错放（"开会放成 XX""活动现场放成 XX"）。
- 想让"说到什么场景就出现什么场景"达到硬保证，而不只是语义倾向。

## override 精修 loop（用户逐句指正时最常用）
用户说「XX秒应该配XX场景/画面」（如「96-102秒应该是首次会议」「208秒应该是庄国栋冷静开会」）时，照此循环：
1. `run --skip-visual --no-render` → 出 `★ 匹配报告.json`（不成片，快）。
2. **定位错配句**：读报告 `segments[]`，按 `output_start`（成片秒）找到用户指的时间点，看该句 `text`（解说）、`clip_start`（现在落的原片秒）、`visual_match_evidence`（现在的画面）。
3. **加 override**：从该句 `text` 里挑一个**独特短语**（别用会撞别句的泛词）作 `contains` 值，`scene` 值填目标场景的 `name`（必须**逐字等于** `scenes[].name`），追加进 `_scene_map.json` 的 `overrides` 数组（格式见下方示例）。一句只需一个短语即覆盖整句所有分镜。
4. 重跑 step 1，复核该句 `clip_start` 是否已落进目标场景的 `ranges`、画面是否对。
5. 反复 2-4 到用户满意 → **去掉 `--no-render` 正式成片**（要保留上次成片先备份到子目录 `旧成片备份/`）。

判断 override 还是 keywords：**某几句特有**（情绪/关系/旁白、无场景词）→ override；**该类场景通用词**（会议的「开会/唱反调」）→ 加进场景 keywords。override 更精准、不误伤别句，是逐句精修主力。

## `_scene_map.json`（放素材文件夹内，每集一份）
```json
{
  "episode": 4,
  "scene_count": 15,
  "overrides": [
    {"contains": "庄国栋表现得礼貌又克制", "scene": "玫瑰加入项目组-历次项目会议"},
    {"contains": "被老板当面拆穿", "scene": "病房-姜雪琼与玫瑰谈策划案"}
  ],
  "scenes": [
    {
      "name": "玫瑰加入项目组-历次项目会议",
      "ranges": [[687,889],[1030,1130],[2254,2326],[2439,2465]],
      "keywords": ["开会","会上","会议上","唱反调","反对","第一天","首次会议","主持会议","新同事","项目会议"],
      "characters": ["黄亦玫","庄国栋"]
    }
  ]
}
```
- `ranges`=原片秒，**多段**（同一场景多次出现/交叉剪辑合并，如食堂三段、会议四段）。放全所有出现段防单段耗尽。
- `keywords`=强判据(×2)，`characters`=弱判据(×1)。缺文件则场景锁定不启用，行为回到纯全片语义匹配。
- `overrides`=**文本精确指定（最高优先）**：一句解说没有场景关键词（情绪/关系句，如"庄国栋表现得礼貌又克制""玫瑰明显动了心"）时，keywords 判据抓不到、会落到全片乱选。用 `{"contains":"解说里的独特短语","scene":"某场景name"}` 把这句**直接钉死**到该场景。**clause-first（2026-07）后**：分类先看每个分镜自己的文字——含 override 短语的分镜命中该 override；同句里**自带场景关键词的分镜**会各归各的场景（一句跨多场景时正是要的）；自己无信号的分镜才回退整句（这时整句里的 override 生效）。所以 override 短语放在**你要钉的那个分镜的文字里**最稳；`scene` 必须精确等于某场景 name。这是"用户指哪配哪、逐句精修"的主力工具。

## 代码位置（已内置）
- `anchored_pipeline.py::_load_scene_map(folder)`：读 `_scene_map.json`，**返回 `(scenes, overrides)`**，支持 `ranges` 多段或 `start/end` 单段。
- `anchored_pipeline.py::_classify_scene(text, scenes, overrides)`：**override 优先**——文本片段命中某 override 就直接返回该场景；否则给解说文字打分 `kw*2+ch`，**要求 ≥1 关键词命中**才锁定（否则回全片）；取最高分场景。
- `anchored_pipeline.py::allocate_visual_all`：**分镜优先分类（clause-first，2026-07）**——先按「本分镜自己的文字」`_classify_scene`（override 或关键词命中就用），自己无场景信号才回退用 `tts_parent_id` 重建的整句文本分类。解决「一句解说跨多场景」（如同一句既讲哥哥食堂线又讲玫瑰会议线）被整句钉到单一场景的张冠李戴。逐镜传 `scene_ranges` 给 allocate；命中的镜头打印 `[场景锁定] 行N-镜M → 「场景名」`。
- `backend/visual_matcher.py::VisualIntervalAllocator.allocate(..., scene_ranges=)`：内层 `_scan(restrict)`。有 `scene_ranges` 就**硬限制**候选窗落在场景区间内；**场景占满(无空闲)才回退全片** `_scan(None)`，绝不崩。语义 embedding + 人物加分照旧，锚定窗 proximity 仅在无场景锁定时生效。

## `--no-render`（只匹配不成片，供审阅）
`autodsj.py run --folder "<素材>" --skip-visual --no-render` → 只跑 TTS+分镜+场景锁定分配，写
`★ 匹配报告.json` / `★ 字幕.srt`，**跳过成片渲染**。用户"先看匹配对不对再成片"时用。
链路：`autodsj.py cmd_run`(args.no_render) → `runner.render(no_render=True)` → `build_pipeline_command` 加 `--no-render` → `anchored_pipeline.main()` 在 `build_timeline` 后 `write_outputs` 然后 return（不 `render_video`）。审阅通过后去掉 `--no-render` 正式成片。

## 给某集建场景图的标准流程
1. 读整集视觉索引时间轴：`_source_visual_index.json` 的 frames 按 `time` 排序，打印 `time [scene] people :: caption`。
2. 用分段脚本得候选段（`scripts/segment_scenes.py "<素材夹>"`）：按 scene 桶+人物+时间跳变>25s 断开，得细碎候选。
3. **结合分集剧情库**（`references/rose-story-episodes.json` 按集取）把细碎候选**合并成有意义的大镜头**并自主命名（如"病房-姜雪琼与苏更生谈玫瑰""天台-黄振华与玫瑰谈心"）。同一场景多次出现/交叉剪辑合并为多 range。
4. 写 `_scene_map.json`，为每场景配 name/ranges/keywords/characters。
5. `dy run --skip-visual --no-render` 出匹配报告，核对场景落点分布 + 用户投诉点是否修正。
6. 迭代关键词/区间/override（见 Pitfalls）→ 用户确认 → 去掉 `--no-render` 成片。
   - 用户逐句指正（"XX秒应该配XX场景"）时：**首选加 override**（`{"contains":该句独特短语,"scene":目标场景}`），
     比调关键词精准可控、不会误伤别句。能加关键词解决的（该类场景通用词）才动 keywords。

## 重跑但不覆盖上次成片
用户要"重跑但保留上次成片"时：先把 `★ 成片.mp4`(+字幕/报告)复制进子目录 `旧成片备份/★ 成片_上一版_<日期>.mp4`，再重跑（新 run 覆盖主目录的 `★ 成片.mp4`，旧版在子目录保住）。子目录不干扰素材检测（跑 `dy status` 确认原片/字幕仍正确识别）。

## 快速迭代精修（本会话第5集实战，4轮从 A66→A75）

第5集 14 场景 + 41 条 override 的 4 轮迭代实录，展示从\"建图→匹配→查错→加 override→重跑\"的完整闭环：

| 轮次 | A/B/C | 关键修复 |
|------|-------|---------|
| v1   | 66/22/5 | 初始 14 场景，35 条 override |
| v2   | 70/14/9 | +庄国栋办公室（帧 214-218，33:53-34:37，夹在姜雪琼办公室和实验室之间被遗漏）；修复\"教追人+崴脚\"→户外约会 |
| v3   | 74/14/5 | 修复\"玫瑰看懂作品→苏更生刮目相看\"→画廊（被句末\"庄国栋发现少人\"整体拖进庄国栋办公室）；修复\"黄振华体面退出\"→户外长椅 |
| v4   | 75/11/7 | 修复\"玫瑰远在上海\"→画廊转正；修复\"做出第一次选择\"→画廊转正 |

每轮只跑 `--skip-visual --no-render`（~2 分钟），不改视觉索引、不改配音。最后一轮通过后再去掉 `--no-render` 正式成片。

## Pitfalls
- **关键词过宽 → 误锁+耗尽**：给会议配"项目组"会把"玫瑰正式进入项目组"（本该配策划书画面）误锁进会议，会议段占满后回退全片抓到食堂路人。关键词要**场景专属**（会议用 开会/会上/唱反调/反对/主持会议，不用泛词"项目组"）。
- **单段场景被订满 → 回退抓错**：把某场景**所有出现段**都放进 ranges（会议 4 段共~400s 而非只 176s），避免占满后 fallback 到错的全片帧。
- **均分会下降是正常的**：硬锁定牺牲原始余弦分换场景正确（第4集 0.80→0.62）。用户明确"场景对 > 分数高"，这是要的取舍，别为了拉高分数放松锁定。
- **按整句而非分镜分类**：分镜clause可能缺关键词，务必用 `tts_parent_id` 重建整句再分类。
- **身份存疑的场景先不配情节关键词**：如"公寓男子"拿不准是谁时，characters 留空、keywords 留空/最小，避免误锁；等用户确认身份再补（本会话：天台谈心=黄振华+玫瑰、公寓=庄国栋赴法前准备）。
- **一句解说跨两个大镜头场景 → 整体被拖进错误场景**（第5集实战）：解说段\"玫瑰不仅看懂了作品…苏更生刮目相看…庄国栋也发现项目组里少了一个人\"——前半讲玫瑰在画廊、末句讲庄国栋在办公室。若只给末句配 override（\"庄国栋发现少人\"→庄国栋办公室），前半也会被整句拖进庄国栋办公室。**修法**：给前半的每个子句都加 override（\"玫瑰不仅看懂了作品\"→画廊、\"一番分析让苏更生刮目相看\"→画廊…），让 clause-first 的逐镜分类把前半锁回画廊。别只改末句。
- **抽象概念旁白即使场景锁对、分数也低**（第5集实战）：旁白\"把兴趣变成解决问题的能力\"\"放弃真正属于自己的机会\"等纯抽象/内心独白，没有任何画面帧能语义匹配（画面只有人站着/走着/看着）。场景锁确保画面在正确物理位置，但 embedding 分必低（0.2-0.3 C 级）。**这是正常的，不用修**——场景对 > 分数高。同理，庄国栋在办公室\"问起玫瑰去了哪里\"也没有视觉上\"问\"的动作（只是坐着看屏幕），分低可接受。
- **视觉索引帧之间的间隙可能夹着新场景**（第5集实战）：庄国栋办公室（33:53-34:37）在视觉索引里只有 5 帧（F214-218），夹在姜雪琼办公室（F201-206）和实验室（F220+）之间。看 plot summary 不会注意到这个场景，必须**逐帧扫描视觉索引时间轴**才能发现。建场景图时必须把全帧时间轴扫一遍，找出所有夹在已知场景之间的独立场景。
